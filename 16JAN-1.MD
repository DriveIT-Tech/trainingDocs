# What is an SSL Certificate?

## What it is (Plain English)

An **SSL/TLS certificate** enables **secure communication** between a user‚Äôs browser and a server.

It ensures:

-   üîê **Encryption** ‚Äì data can‚Äôt be read in transit
    
-   üßæ **Authentication** ‚Äì you‚Äôre talking to the real server

## What problem it solves

### Without SSL:

-   Passwords travel as plain text
    
-   Anyone on the network can read or modify traffic
    

### With SSL:

-   Data is encrypted using public/private keys
    

----------

## How it works (High-level)

1.  Browser connects to `https://example.com`
    
2.  Server sends its SSL certificate
    
3.  Browser verifies the certificate (CA trust)
    
4.  Secure encrypted session is established
    

----------

## Real-world examples

-   Online banking
    
-   Login pages
    
-   APIs exchanging tokens

# Apache vs Nginx vs Tomcat (with Use Cases)

## Apache HTTP Server

### What it is

A **general-purpose web server**.

### Strengths

-   Simple configuration
    
-   Huge ecosystem
    
-   `.htaccess` support
    

### Weakness

-   Heavy under high concurrent load
    

### Best use cases

-   Legacy applications
    
-   Shared hosting
    
-   PHP-based apps (WordPress)
    

----------

## Nginx

### What it is

A **high-performance web server and reverse proxy**.

### Strengths

-   Handles thousands of connections efficiently
    
-   Low memory usage
    
-   Excellent as reverse proxy / load balancer
    

### Weakness

-   Less flexible per-directory configs
    

### Best use cases

-   Front-end web server
    
-   Reverse proxy
    
-   API gateways
    
-   Kubernetes ingress
    

----------

## Tomcat

### What it is

A **Java application server**.

### Important distinction

‚ùó **Tomcat is not a general web server** like Apache or Nginx.

### Strengths

-   Runs Java apps (WAR files)
    
-   Servlet & JSP support
    

### Weakness

-   Not optimized for static content
    
-   Not ideal as edge-facing server
    

### Best use cases

-   Java web applications
    
-   Used behind Nginx / Apache
    

----------

### Typical Enterprise Setup

`Internet
   |
 Nginx (SSL, Load Balancing)
   |
 Tomcat (Java App)`

# SSH Command Examples (Must-Know)

### Basic SSH Login

`ssh user@server_ip` 

### SSH with Key

`ssh -i key.pem user@server_ip` 

### Copy file to server

`scp file.txt user@server:/tmp/` 

### Copy file from server

`scp user@server:/tmp/file.txt .` 

### Run command remotely

`ssh user@server "df -h"` 

### SSH Port Forwarding

`ssh -L 8080:localhost:80 user@server` 

### Check SSH service

`systemctl status sshd`
